This run matches the preliminary 2026-07-22 configuration: one closed-loop
run per fixture and scenario, 10 seconds, 64 concurrent HTTP/1.1 connections,
and stable fixture order. It is useful for a local before/after comparison but
is not a publishable benchmark.

## dart_orpc comparison with the previous matched run

| Scenario | 2026-07-22 | 2026-07-23 | Change |
| --- | ---: | ---: | ---: |
| Plaintext | 46,801 req/s | 68,229 req/s | +45.8% |
| JSON | 47,193 req/s | 65,611 req/s | +39.0% |
| Echo | 56,798 req/s | 56,167 req/s | -1.1% |

Plaintext and JSON improved substantially in this matched run, while echo is
effectively unchanged. The shared-container implementation primarily changes
startup and provider lifecycle rather than the per-request hot path, so this
single before/after result does not establish that DI caused the throughput
increase. Randomized repeated runs of both revisions are needed for causal
attribution.
